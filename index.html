Writing

How to build a website

I'm an advocate for Agile principles and methodologies. I believe that communication is key to creating great products. My process is always evolving and I love to experiment with new technologies. I've documented my process here in the hopes of teaching more people the wonderful art of web development.

#1. Communicate ideas in pen and paper.

I use pocket sized notebooks with gridded paper like Field Notes to work on content or information architecture. This is also a good time to list any features you expect to implement. List any questions you want answered before moving on.

#2. Establish look in Sketch.

Size, color, fonts, and hierarchy are much easier to find in a graphic editing program like Sketch than it is in the browser. Sketch is an amazing UI design tool that allows you to experiment with ideas quickly. It also great for exporting assets, so finding pictures and icons at this stage is also important.

#3. Develop Front-End in HTML/CSS/JS.

Create a static version of the website or interface based on the design. Do not worry about saving or retrieving data yet, just establish how the data will look. I use Atom as a text editor alongside iTerm running ZSH. I use Gulp to preform tasks such a compiling SCSS or running a local server. I follow front-end code guidelines as listed on workmanship.io.

#4. Connect and deploy site.

I am a fan of Node.js based frameworks like Express.js, since JavaScript can then be used for the front-end and back-end. Meteor.js has a cool way of storing and accessing information using DDP, so I like to use it for real-time applications. I use Digital Ocean to host most of my websites. If its a simple page, i'll use Github Pages instead.
