03: Process

I'm an advocate for Agile principles and methodologies. I believe that communication is key to creating great products. My process is always evolving, I love to experiment with new tools and technologies. I've documented my process here in the hopes of teaching more people the wonderful art of web development.

#1. Communicate ideas in pen and paper.

I really enjoy gridded pocket sized notebooks like Field Notes or Word Notebooks and a nice mechanical pencil. This is a great time to work on content or information architecture. It is also a good time to list any features you expect to implement. List any questions you want answered before moving on.

#2. Establish look in Sketch.

Size, color, fonts, and hierarchy are much easier to find in a graphic editing program like Sketch than it is in the browser. Sketch is an amazing UI design tool that allows you to experiment with ideas quickly. It is faster to place content directly in HTML than in Sketch, so only focus on visual elements. Do only enough work necessary in Sketch to communicate an idea or establish an asethetic.

#3. Develop Front-end in HTML/CSS/JS.

Take the design and create a static version of the website or interface. Do not worry about saving or retrieving data yet, just establish how the data will look. I use many tools in this process, such as Atom text editor and Gulp task runner. Stylesheet preprocessors such as Scss are useful, but not always necessary. I avoid frameworks such as Bootstrap, and prefer to build most things from scratch.

#4. Connect and deploy site.

Most websites I have created are static or have been passed on to other developers to make dynamic. I am a fan of Node.js based frameworks like Express.js, since javascript can then be used for the front-end and back-end. I think that Meteor.js has an interesting way of storing and accessing information, and prefer to use it for real-time applications. I use Digital Ocean to host dynamic sites or Github Pages to host static sites. Continuous intergration with Deploybot alongside a Bitbucket repository is a small slice of heaven.
